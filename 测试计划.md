# 形码大师 — 测试计划

## 1. 测试概述

### 1.1 测试目标
验证形码大师App v1.0.0 MVP版本的功能正确性、稳定性和兼容性。

### 1.2 测试范围
- 核心练习功能（单字/词组/句子练习）
- 实时反馈与数据统计
- 字根表与查询功能
- 错字本功能
- 设置功能（码表切换、主题切换）
- 统计功能

### 1.3 测试环境
- Android Studio Hedgehog | 2023.1.1
- Android Emulator: Pixel 7 API 34
- 物理设备: Android 13/14
- Kotlin 1.9.22
- Gradle 8.2

## 2. 测试策略

| 测试类型 | 方法 | 工具 | 覆盖率目标 |
|----------|------|------|------------|
| 单元测试 | 自动化 | JUnit4 | 60% |
| 集成测试 | 自动化 | Espresso | 核心流程 |
| 功能测试 | 手动 | 测试用例 | 100% |
| 兼容性测试 | 手动 | 多设备 | Android 8-14 |
| 性能测试 | 手动/自动化 | Android Profiler | 关键场景 |

## 3. 测试进度

| 阶段 | 时间 | 任务 |
|------|------|------|
| 单元测试 | Day 1 | 工具类、ViewModel单元测试 |
| 集成测试 | Day 2 | 数据库、Repository测试 |
| 功能测试 | Day 3-4 | 按测试用例执行 |
| 兼容性测试 | Day 5 | 多设备验证 |
| 回归测试 | Day 6 | Bug修复后验证 |

## 4. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 码表数据不完整 | 高 | 使用最小数据集验证核心逻辑 |
| Compose UI测试不稳定 | 中 | 增加等待时间，使用IdlingResource |
| Room数据库迁移 | 低 | 使用fallbackToDestructiveMigration |

## 5. 通过标准

- 所有P0功能测试用例通过
- 无阻塞性Bug
- 崩溃率 < 0.1%
- 启动时间 < 3秒
